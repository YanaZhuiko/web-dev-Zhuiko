1. When HTML5 became a standard?
HTML5 became a standard on October 28, 2014.
2. What were the main reasons for developing HTML5?
The main reasons included improving multimedia support, enhancing web application capabilities, and introducing advanced features to the markup language.
3. Difference between W3C HTML5 and WHATWG HTML Living Standard?
W3C HTML5 represents stable versions released by W3C, while WHATWG HTML Living Standard is continuously updated, reflecting ongoing HTML development.
4. What is the minimal HTML page code?
The minimal HTML page code includes the DOCTYPE declaration, HTML, head with a title, and a body with content.
5. Comment on semantic and presentational markup. Why presentational markup is considered deprecated now?
Semantic markup focuses on meaning (e.g., <header>, <article>), while presentational markup deals with appearance (e.g., <font>, <b>). Presentational markup is considered deprecated because it mixes structure and style, making it less maintainable and impacting accessibility negatively.
6. How can you check browser support for a particular tag or attribute?
To check browser support, you can use online resources like MDN Web Docs or CanIUse.com. Dynamic checks can also be performed using JavaScript or CSS.
7. What are the audio and video formats officially supported by HTML?
Officially supported audio formats are MP3, WAV, and OGG. Video formats include MP4, WebM, and OGG. Using other formats may result in browsers being unable to play the content.
